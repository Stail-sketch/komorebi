# 木漏れ日遊園地 — Night1 実装指示書（Claude Code用）

## プロジェクト概要
FNAFライクなホラーブラウザゲーム。HTML/CSS/JS単体で実装する。
ゲームエンジン不使用。画像素材は完成済み。

## ゲームシステム詳細

### 基本ルール
- **ゲーム時間**: ゲーム内 12:00AM〜6:00AM（リアル約5分 = 300秒）
  - 1ゲーム内時間 = リアル50秒
- **電力**: 100%からスタート。0%で全システム停止→ゲームオーバー
- **クリア条件**: 6:00AMまで生存

### 電力消費
| 状態 | 消費率（/秒） |
|------|-------------|
| 基本消費（何もしていない） | 0.05%/秒 |
| カメラ表示中 | +0.05%/秒 |
| 左シャッター閉 | +0.15%/秒 |
| 右シャッター閉 | +0.15%/秒 |
| 両方閉じ | +0.30%/秒 |

※ 仮の値。プレイテストで調整前提。

### 時刻表示
- 300秒 ÷ 6時間 = 50秒/時間
- 12AM → 1AM → 2AM → ... → 6AM
- 6AM到達でクリア演出

## キャラクター行動パターン（Night1）

### キツネ団長（最重要・常時管理が必要）
- **初期位置**: CAM2（からくり時計セット）
- **ギミック**: 時計ゲージ
  - ゲージは100からスタート
  - **常に減少し続ける**（約0.5/秒。仮値）
  - CAM2を表示中にクリック連打するとゲージ回復（1クリック = +3。仮値）
  - CAM2以外を見ている間もゲージは減り続ける
  - ゲージが0になった瞬間 → **即ゲームオーバー（シャッター無効）**
- **合図**: ゲージが30%以下になるとBGMにチクタク音が混じる
- **シャッター**: **無効**（時計巻きのみが対策）

### かあ博士
- **初期位置**: CAM4（研究室セット）
- **行動サイクル**:
  1. CAM4で待機（初回: ゲーム開始後60〜90秒でランダムに移動開始）
  2. CAM6（バックステージ）に移動
  3. CAM7（左通路）またはCAM8（右通路）にランダムで移動
  4. 扉前に到達（カメラ映像では見えない。扉のシルエットで確認）
  5. 扉前で数秒待機後、シャッターが開いていれば侵入→ゲームオーバー
  6. シャッターが閉まっていれば撃退→CAM4に戻る→サイクル繰り返し
- **移動間隔**: 各ステップ間 8〜15秒（ランダム。仮値）
- **扉前待機**: 5〜10秒（この間にシャッターを閉じれば撃退）
- **シャッター**: **有効**
- **カメラ確認**: CAM7にいれば左扉、CAM8にいれば右扉に来る

## ゲームオーバー演出
1. 画面が一瞬暗転
2. ジャンプスケア画像を0.5〜1秒表示（該当キャラのjumpscare画像）
   - キツネ団長: kitsune_jumpscare.png
   - かあ博士: kaa_jumpscare.png
3. 大きめの効果音（ジャンプスケア音）
4. 画面暗転→リトライ画面

## クリア演出（6:00AM到達）
1. 画面が明るくなる
2. 「6:00 AM」大きく表示
3. 短い成功音
4. 数秒後にクリア画面→Night2へ（Night2は未実装なので仮画面）

## 音響設計（仮実装）
Web Audio APIまたは`<audio>`タグで実装。
Night1では最低限以下を用意：
- **BGM**: 低いアンビエント（ループ）。なければ無音でもOK
- **カメラ切替音**: ザッという短いノイズ
- **シャッター開閉音**: ガシャンという金属音
- **時計の巻き音**: カチカチ（連打に合わせて）
- **チクタク警告音**: ゲージ30%以下で混じる
- **ジャンプスケア音**: 大きめの不協和音
- **6AM チャイム**: 成功音

→ まず音なしでゲームロジックを完成させ、後から音を差し込む方針でOK。

## 実装の優先順位

### Step 1: 画面構造とカメラ切替
- night1.htmlの基本レイアウト
- 8台のカメラ切替（背景画像の差し替え）
- カメラ表示/非表示の切り替え
- 左右シャッターの開閉UI

### Step 2: ゲームタイマーと電力
- 12AM→6AMの時刻進行
- 電力ゲージ（シャッター閉で消費増）
- 電力0でゲームオーバー
- 6AM到達でクリア

### Step 3: キツネ団長（時計ギミック）
- 時計ゲージの常時減少
- CAM2表示中のクリック連打で回復
- ゲージ0で即死（ジャンプスケア→ゲームオーバー）
- チクタク警告（ゲージ低下時）

### Step 4: かあ博士（ランダム移動）
- 移動ロジック（CAM4→CAM6→CAM7orCAM8→扉前）
- カメラに映る/映らないの制御
- 扉前到達時のシルエット表示
- シャッター判定（開→ゲームオーバー / 閉→撃退）

### Step 5: ジャンプスケアとクリア演出
- ゲームオーバー時のジャンプスケア画像表示
- クリア時の6AM演出
- リトライ機能

### Step 6: 画像差し込みと調整
- カメラ背景にcam1〜8.pngを配置
- キャラ画像をカメラ映像上に重ねて表示
- 扉前シルエットの表示

## 技術メモ
- **フレームワーク不使用**。素のHTML/CSS/JSで書く
- **requestAnimationFrame** または **setInterval** でゲームループ
- **状態管理**: グローバルなgameStateオブジェクトで一元管理
- **localStorage**: Night1クリアフラグの保存（後のNight解放用）
- 画像がまだ差し込めない段階では、色付きの矩形やプレースホルダーで代替してOK
- **レスポンシブ不要**。PC横画面（16:9程度）を想定

## gameState 設計例
```javascript
const gameState = {
  night: 1,
  time: 0,           // 0〜300秒
  power: 100,        // 0〜100%
  gameOver: false,
  cleared: false,
  
  camera: {
    active: false,    // カメラ表示中か
    current: 1,       // 現在のカメラ番号 1〜8
  },
  
  shutters: {
    left: false,      // true = 閉じている
    right: false,
  },
  
  clockGauge: 100,    // キツネ団長の時計ゲージ 0〜100
  
  characters: {
    kitsune: {
      position: 'cam2',  // Night1では基本CAM2固定（時計ギミック専用）
    },
    kaa: {
      position: 'cam4',  // cam4 → cam6 → cam7/cam8 → door_left/door_right
      nextMoveTime: 0,   // 次の移動タイミング
      atDoor: false,
      doorSide: null,     // 'left' or 'right'
      doorArrivalTime: 0, // 扉前に到着した時刻
    },
  },
};
```

## 注意事項
- Night1だけを完成させる。Night2以降のキャラ（うっきち・ぽたまる・試作きつね）は実装しない
- 数値バランスは全て仮。後からプレイテストで調整する前提
- まず画像なし（プレースホルダー）で動作するゲームロジックを完成させ、その後画像を差し込む
- ジャンプスケアは画像表示のみでOK。音は後回し
