## Night2: うっきち追加

### 新キャラ: うっきち（サル型獣人）
- **初期位置**: CAM5（小道具倉庫）
- **ギミック**: カメラに映った通路と**反対**の扉から来る
  - CAM7（左通路）に映っている → **右扉**から侵入
  - CAM8（右通路）に映っている → **左扉**から侵入
- **移動経路**: CAM5 → CAM6 → CAM7またはCAM8 → 反対側の扉前
- **対処法**: カメラで確認した通路の**反対側**のシャッターを閉める
- **シャッター**: 有効
- **ジャンプスケア**: ukkichi_jumpscare

### Night2の登場キャラ
- キツネ団長（Night1と同じ）
- かあ博士（Night1と同じ、やや攻撃性UP）
- うっきち（新規）

### Night2の難易度パラメータ例
```javascript
night2: {
  clockGaugeDecay: 0.6,        // 時計ゲージ減少速度（Night1: 0.5）
  kaa: {
    moveInterval: [7, 13],      // 移動間隔（Night1: [8, 15]）
    doorWaitTime: [5, 9],       // 扉前待機
  },
  ukkichi: {
    moveInterval: [10, 18],     // 初登場なので遅め
    doorWaitTime: [6, 10],
  },
}
```

## Night3: ぽたまる追加

### 新キャラ: ぽたまる（タヌキ型獣人）
- **初期位置**: CAM3（ダンスフロア）
- **ギミック**: ダンス音楽が流れた瞬間に一気に扉前に到達（盗塁）
  - **音楽A** → 右通路経由で**右扉前**に到達
  - **音楽B** → 左通路経由で**左扉前**に到達
- **対処法**: 音楽が流れた瞬間に対応するシャッターを閉める。反射神経勝負。
- **シャッター**: 有効
- **ジャンプスケア**: potamaru_jumpscare

### ぽたまるの音楽ギミック実装
- ランダムなタイミング（30〜60秒間隔。仮値）で音楽A or Bが流れる
- 音楽が流れてから扉前到達までの猶予: 2〜3秒（仮値）
- 猶予内にシャッターを閉めれば撃退 → CAM3に戻る
- 閉められなければゲームオーバー
- **音楽AとBは聴き分けられる必要がある**（音程・リズムが明確に違う短いフレーズ）
  - 音楽素材は後で用意。仮実装では高音ビープ=A（右）、低音ビープ=B（左）で代替

### Night3の登場キャラ
- キツネ団長
- かあ博士（攻撃性UP）
- うっきち（攻撃性UP）
- ぽたまる（新規）

### Night3の難易度パラメータ例
```javascript
night3: {
  clockGaugeDecay: 0.7,
  kaa: {
    moveInterval: [6, 11],
    doorWaitTime: [4, 8],
  },
  ukkichi: {
    moveInterval: [8, 15],
    doorWaitTime: [5, 8],
  },
  potamaru: {
    musicInterval: [30, 60],    // 音楽発生間隔
    rushTime: 3,                // 音楽から扉到達までの猶予（秒）
  },
}
```

## Night6: 試作きつね追加（隠しNight）

### 前提条件
- Night5クリア済み
- ARG謎解きで試作きつねの情報（出現条件・カメラ切替順）を入手済み
  （ARG未実装のため、仮で直接Night6にアクセスできるようにする）

### 新キャラ: 試作きつね（キツネ型・隠しキャラ）
- **Night6専用**
- **ギミック**: 特定の条件で突然出現 → 出現から数秒で即死
- **シャッター**: **無効**
- **対処法**: ARGで入手した「特定のカメラ切替順」を実行することでのみ撃退
- **ジャンプスケア**: shisaku_jumpscare

### 試作きつねの仮実装
出現条件・カメラ切替順は未定のため、仮で以下の仕様にする:
- **出現条件**: ランダムなタイミング（60〜120秒間隔）で出現
- **出現合図**: 画面にノイズが走る、または特殊な音が鳴る
- **猶予**: 出現から5秒以内に撃退しないと即死
- **仮の撃退方法**: CAM1→CAM3→CAM5→CAM7の順にカメラを切り替える
  （ARG設計時に正式な順番を決定）

## 全Night共通の実装ルール

### Night選択画面
- 各Nightはクリア済みフラグで順番に解放
- localStorage: night1_cleared, night2_cleared, ... night5_cleared
- Night6は特殊（ARG経由。仮でnight5_clearedがあれば表示）

### 各Nightの開始フロー
1. 黒画面に「Night X」表示
2. 「Click to Start」
3. クリック → フェードアウト → ゲーム開始 + BGM再生

### ゲームオーバー画面
- ジャンプスケア（該当キャラの画像+音声）
- 暗転 → 「Game Over」表示
- 「Retry」ボタン → 同じNightをやり直し

### クリア画面（Night1〜4）
- 6AM表示 + clear_yay再生
- 数秒後にフェード → Night選択画面に戻る
- 次のNightが解放される

### 実装順序
1. Night2（うっきち追加）
2. Night3（ぽたまる追加 + 音楽ギミック）
3. Night4（パラメータ強化のみ）
4. Night5（パラメータMAX + メインエンディング演出）
5. Night6（試作きつね + 真エンディング演出）

### 各Nightの実装ステップ
Step A: 新キャラの行動ロジック追加
Step B: 難易度パラメータ設定
Step C: ジャンプスケア画像・音声の紐付け
Step D: クリア判定とフラグ保存
Step E: 演出（Night5・Night6のみ）
Step F: プレイテスト・バランス調整
